Managing Godot projects across a team is messier than it should be. Different contributors run different engine versions. Addons are committed wholesale into the repository or documented in a README that nobody reads. There is no standard way to reproduce the full project environment from scratch.

{{ tool_name() }} fixes this.

## What is {{ tool_name() }}?

{{ tool_name() }} is a project manager for Godot games, inspired by [uv](https://github.com/astral-sh/uv) for Python. It gives every project a `ggg.toml` file that declares two things:

1. **The Godot version** the project requires
2. **A list of addon dependencies** referenced as git coordinates

From there, a single command - `{{ cli() }} sync` - downloads the right Godot binary and installs every declared addon at the exact revision pinned in `ggg.lock`. No more README instructions, no more committed vendor trees, no more "works on my machine" engine mismatches.

## Core Concepts

**`ggg.toml`** lives at the project root alongside `project.godot`. It is the source of truth for the project's environment:

```toml
[project]
godot = "4.3.0"

[[dependency]]
name   = "gut"
git    = "https://github.com/bitwes/Gut.git"
rev    = "v9.3.0"
target = "addons/gut"
```

**`ggg.lock`** is generated by `{{ cli() }} sync`. It records the exact commit SHA resolved for every dependency so that installs are bit-for-bit reproducible. Commit it to your repository.

**The cache** lives in a platform-appropriate directory (`~/.local/share/ggg/` on Linux, `%APPDATA%\ggg\` on Windows). Godot binaries and git checkouts are stored there and shared across all your projects.

## CLI Overview

| Command | What it does |
|---|---|
| `{{ cli() }} init` | Create a `ggg.toml` in the current directory |
| `{{ cli() }} sync` | Download Godot + install all dependencies |
| `{{ cli() }} edit` | Open the project in the pinned Godot editor |
| `{{ cli() }} run [-- <args>]` | Run Godot against the project |
| `{{ cli() }} add <git-url>` | Add a new dependency |
| `{{ cli() }} remove <name>` | Remove a dependency |
| `{{ cli() }} self update` | Update {{ tool_name() }} itself |
